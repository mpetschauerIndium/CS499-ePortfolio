CS499 Capstone Reflection

My CS499 capstone represents the progression from completing course-based programming assignments to building, evaluating, and refining a professional full-stack application. Through the Travlr Getaways Admin project, I demonstrated growth in software engineering, algorithms, databases, security, and technical communication.

Download Professional Self Assessment

Coursework Reflection

My computer science coursework provided the foundation for designing applications that are structured, testable, and maintainable. Earlier classes introduced programming fundamentals, data structures, databases, web development, and secure coding practices. In the capstone, I applied those concepts together by enhancing the original CS465 TRAVLR application into a more complete administrative system.

Collaboration

The capstone required the same habits used in professional development: reviewing existing code, identifying risks, planning enhancements, documenting decisions, and refining the work through feedback. I treated the project as an existing codebase that needed careful improvement rather than a blank-slate rewrite.

Communication

This portfolio demonstrates technical communication through artifact summaries, enhancement narratives, a code review summary, and project screenshots. The goal is to explain not only what was built, but why specific engineering decisions improved security, maintainability, scalability, and data integrity.

Software Engineering

The software engineering enhancement refactored the application into clearer backend and frontend responsibilities. Express routes, controllers, services, middleware, utilities, and Mongoose models now have more focused roles, while the Angular client uses services, guards, interceptors, and routed components to support authenticated administrative workflows.

Algorithms

The algorithms and data structures enhancement improved how trip data is normalized, identified, retrieved, and rendered. Pagination, deterministic sorting, unique trip-code generation, safe identifier handling, derived Angular state, and stable list tracking all support a more predictable and scalable trip management experience.

Database

The database enhancement strengthened MongoDB persistence with stricter Mongoose schemas, validation rules, timestamps, indexes, sanitized filters, lean reads, validator-backed updates, and protected user password fields. These changes help preserve data quality and reduce risk as the application grows.

Security

Security was a major focus of the capstone. The application now supports administrator login, JWT authentication, Bearer token middleware, protected create/update/delete routes, request validation, CORS configuration, and salted PBKDF2 password hashes instead of plaintext passwords.

Career Goals

My goal is to continue growing as a software developer who can contribute to full-stack applications, backend APIs, data-driven systems, and secure administrative tools. The Travlr capstone shows the type of work I want to build professionally: practical, maintainable software that connects user needs with reliable technical implementation.
